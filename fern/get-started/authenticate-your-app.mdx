Access Tokens provide secure client-side authentication for Cartesia's APIs without exposing your API key.

## Overview

- **Server-side Generation**: Tokens must be generated on your backend server
- **Short-lived Security**: Configurable lifetime up to 1 hour
- **Granular Permissions**: Control access to specific Cartesia services

## Prerequisites

Before implementing Access Tokens:

1. Set up a secure backend server
2. Configure your server with a Cartesia API key
3. Implement user authentication in your application
4. Establish secure client-server communication

### Available Grants

Currently supported endpoints per grant:

#### TTS Grant

- `/tts/bytes` - Synchronous TTS generation
- `/tts/sse` - Server-sent events for streaming
- `/tts/websocket` - WebSocket-based streaming

> **Coming Soon**: Additional grants for `/voices`, `/voice-changer`, and other services

## Implementation Guide

### 1. Token Generation (Server-side)

Make a POST request to generate a new access token:

```typescript
POST https://api.cartesia.ai/tokens

// Headers
{
  "Content-Type": "application/json",
  "Cartesia-Version": "2024-11-13",
  "X-API-Key": "<your_api_key>"
}

// Body
{
  "grants": {
    "tts": boolean
  },
  "expiresIn": number  // milliseconds
}
```

#### Example Implementation

```typescript
const response = await fetch("https://api.cartesia.ai/tokens", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Cartesia-Version": "2024-11-13",
    "X-API-Key": apiKey,
  },
  body: JSON.stringify({
    grants: { tts: true },
    expiresIn: 60000, // 1 minute
  }),
});

const { token } = await response.json();
```

For detailed API specifications, see the [Token API Reference](/api-reference/identity-and-access/token).

### 2. Token Storage (Client-side)

Implement secure token storage using:

```typescript
// Set HTTP-only cookie
app.use(
  cookieSession({
    name: "session",
    httpOnly: true,
    secure: true,
    sameSite: "strict",
    maxAge: 60000, // Match token expiration
  })
);
```

### 3. Making Authenticated Requests

```typescript
const response = await fetch("https://api.cartesia.ai/tts/bytes", {
  headers: {
    Authorization: `Bearer ${accessToken}`,
    "Content-Type": "application/json",
  },
  // ... request configuration
});
```

## Security Best Practices

### Essential Guidelines

- ✅ Generate tokens server-side only
- ✅ Use short token lifetimes (minutes rather than hours)
- ✅ Implement automatic token refresh
- ✅ Store tokens in HTTP-only cookies
- ✅ Enable secure and SameSite cookie flags

### Security Don'ts

- ❌ Never store tokens in localStorage/sessionStorage
- ❌ Never expose tokens in URLs
- ❌ Never log tokens or display them in the UI
- ❌ Never transmit tokens over non-HTTPS connections

### Token Lifecycle Management

1. Generate new token upon user authentication
2. Implement automatic refresh before expiration
3. Revoke tokens on user logout
4. Handle token expiration gracefully
